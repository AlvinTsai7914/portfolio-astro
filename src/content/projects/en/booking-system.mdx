## Project Overview

An online booking system built for a restaurant chain. Customers select tables, time slots, and add notes; restaurants have a calendar view, seat management, and customer data. On booking, the system automatically sends LINE notifications and writes to the restaurant's Google Calendar.

## My Role

Full-stack development, responsible for:

- Next.js App Router architecture (customer + admin)
- Prisma schema design (restaurants / tables / time slots / bookings)
- LINE Messaging API integration (webhook + push message)
- Google Calendar OAuth and two-way sync

## Technical Highlights

### Slot Conflict Handling

Used a "tables × time slices" 2D array to compute available slots, avoiding the performance issues of nested loops on large datasets. When creating bookings, PostgreSQL's `EXCLUDE` constraint blocks conflicts at the DB layer too — double protection.

### Notification Queue

LINE pushes can't block customer requests on failure, so notifications are decoupled into a BullMQ queue for async processing. On failure, retry 3 times automatically; if still failing, log the error for admin review.

## Lessons Learned

Doing it for real revealed how complex "business logic" really is — a single rule like "restaurant takes a sudden break" spans booking creation, notification copy, refund flows, and a dozen other touchpoints. Writing code was only 30%; discussing edge cases with the client took the rest.
