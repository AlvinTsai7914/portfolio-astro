## Project Overview

A practice project for learning CRDT (Conflict-free Replicated Data Type). Built a multi-user collaborative Markdown editor where multiple users editing the same document simultaneously have all their edits merged automatically without conflicts.

## My Role

Solo development, focusing on:

- CodeMirror 6 integration (syntax highlighting, line numbers, Markdown plugin)
- Yjs CRDT with WebSocket Provider
- Multi-user cursor and selection sync display
- Version history and recovery

## Technical Highlights

### CRDT Logic

Every edit is encoded by Yjs as an immutable update binary and broadcast over WebSocket to other connections. Each client merges incoming updates into its own doc; the math guarantees identical results regardless of merge order, so no central server arbitration is needed.

### Cursor Sync

Each user gets a uniquely-colored cursor. Local selection changes are broadcast via the awareness protocol; remote cursors are rendered via CodeMirror's decoration API as inline widgets, with name tags fading in/out via CSS animation to reduce visual noise.

## Lessons Learned

CRDT is elegant but hard to understand. Building it taught me "data structure determines collaboration capability" — pick OT (Operational Transform) at the start and you're locked into a central server. Yjs gives the freedom of "offline-first + eventual consistency," but that also means dealing with all the strange bugs of asynchronous merging.
