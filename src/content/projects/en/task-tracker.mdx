## Project Overview

A self-use practice project. Built an offline-first task management tool usable without network access, auto-syncing once connected. Focus on IndexedDB operations and Service Worker offline strategies.

## My Role

Solo. Tech choices intentionally challenged me:

- Zustand instead of Redux to practice lighter state management
- Direct IndexedDB (no Dexie or other wrappers) to feel the raw API
- Workbox-configured Service Worker offline strategies
- Sync conflict resolution (last-write-wins + user choice)

## Technical Highlights

### Offline-First Architecture

All reads/writes hit IndexedDB first with immediate UI updates; the sync layer pushes changes to the backend in the background, logging failures for retry. Users always feel the app is instant — the network is just an "optimization," not a dependency.

### Service Worker Strategies

Different resource types get different cache strategies: HTML uses Network First (stays fresh); CSS/JS uses Stale While Revalidate (fast and fresh); API uses Network Only with fallback to Cache (offline returns stale data). The strategies are written as a configurable config file to avoid unmaintainable Service Worker code.

## Lessons Learned

"Offline-first" sounds simple but is complex in practice — it's not just "works without network," but flipping the entire data flow from "network is primary, local is cache" to "local is primary, network is sync." Only after building it did I understand why such tools usually have large teams: the cases to handle are genuinely many.
