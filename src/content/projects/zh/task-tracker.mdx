## 專案概述

自用練習作品。建立一個離線優先的任務管理工具，可以在沒有網路的環境下使用，連線後自動同步。重點放在 IndexedDB 操作與 Service Worker 的離線策略。

## 我的角色

獨立完成。技術選擇刻意挑戰自己：

- Zustand 取代 Redux 練習更輕的 state management
- IndexedDB 直接操作（不用 Dexie 等 wrapper）感受底層 API
- Workbox 配置 Service Worker 離線策略
- 同步衝突處理（last-write-wins + 使用者選擇）

## 技術亮點

### 離線優先架構

所有讀寫先走 IndexedDB，UI 立即更新；同步層在背景把變更推上後端，失敗則記錄待重試。使用者感覺永遠都是即時的，網路只是「優化項」而非依賴項。

### Service Worker 策略

不同資源類型用不同快取策略：HTML 走 Network First（保持最新）、CSS/JS 走 Stale While Revalidate（快又新）、API 用 Network Only fallback to Cache（離線回退舊資料）。把策略寫成可組態的配置檔，避免 Service Worker 程式碼難維護。

## 學到的事

「離線優先」說起來簡單做起來複雜——不只是「斷網能用」，而是把整個資料流從「網路是主、本地是快取」翻轉成「本地是主、網路是同步」。寫完才理解為什麼這類工具往往團隊規模都很大：要照顧的 case 真的多。
